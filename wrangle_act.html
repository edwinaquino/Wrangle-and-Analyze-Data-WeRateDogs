Introduction¶

In this project, we will be gathering data from the WeRateDogs Twitter archive. The goal for this project is to wrangle WeRateDogs Twitter data to create interesting and trustworthy analyses and visualizations.

Real-world data rarely comes clean. We will be using Python and its libraries to gather data from a variety of sources and in a variety of formats. Then, we will assess its quality and tidiness. After that, we will clean the data. This is called data wrangling! On this project we will document our wrangling efforts in this Jupyter Notebook, plus showcase them through analyses and visualizations using Python and its various libraries.

The dataset that we will be wrangling, analyzing and visualizing is the tweet archive of Twitter user @dog_rates, also known as WeRateDogs. WeRateDogs is a Twitter account that rates people's dogs with a humorous comment about the dog. WeRateDogs downloaded their Twitter archive and sent it to Udacity via email exclusively for us to use in this project. This archive contains basic tweet data (tweet ID, timestamp, text, etc.) for all 5000+ of their tweets as they stood on August 1, 2017.

Part I - Gathering Data¶

Go Back To The Top

Gathering Data for this Project will involved the following three parts:

  1. The WeRateDogs Twitter archive. We will download this file manually from the following URL: https://d17h27t6h515a5.cloudfront.net/topher/2017/August/59a4e958_twitter-archive-enhanced/twitter-archive-enhanced.csv

  2. The tweet image predictions, i.e., what breed of dog (or other object, animal, etc.) is present in each tweet according to a neural network. This file (image_predictions.tsv) is hosted on Udacity's servers and will be downloaded programmatically using the Requests library from the following URL: https://d17h27t6h515a5.cloudfront.net/topher/2017/August/599fd2ad_image-predictions/image-predictions.tsv

  3. Each tweet's retweet count and favorite ("like") count at minimum, and any additional data we find interesting. Using the tweet IDs in the WeRateDogs Twitter archive, we will query the Twitter API for each tweet's JSON data using Python's Tweepy library and store each tweet's entire set of JSON data in a file called tweet_json.txt file. Each tweet's JSON data will be written to its own line. Then read this .txt file line by line into a pandas DataFrame with (at minimum) tweet ID, retweet count, and favorite count. Note: Our Twitter API keys, secrets, and tokens will not be included in this notebook for security purposes.

Part III - Cleaning Data¶

Go Back To The Top

The 3rd step in data wrangling involves cleaning. Unclean Data: Dirty vs. Messy. There are two types of unclean data:

  1. Dirty data, also known as low quality data. Low quality data has content issues.
  2. Messy data, also known as untidy data. Untidy data has structural issues.

Cleaning is resolving quality and tidiness issues. We will identify these issues below.

Quality:¶

We must performed atleast 8 quality issues. These are the issues will be evaluating to enhance the quality of our data:

  1. Remove retweets
  2. Remove duplicate tweet_id's
  3. Remove tweets with no images
  4. Remove the columns included in the retweets since they are not needed
  5. Remove duplicate column date_time which contains the same values as the timestamp column from the API
  6. Remove image unnecessary predictions columns
  7. Find and remove duplicate values with unnecessary information
  8. Find empty names and correct the dirty data
  9. Get Dogs gender column from text column
  10. Change the 'NaN' records to None Type
  11. Change column names to be more accurate
  12. Convert each column to its appropriate data type

# The purpose of this loop is to help us identify some of the conditions listed and make the corrections
dog_names = []

for text in df_clean['text']:
    # Starts with 'This is ' and the first letter of the name is either lowercase or uppercase
    if text.startswith('This is ') and re.match(r'[a-z][A-Z].*', text.split()[2]):
        dog_names.append(text.split()[2].strip(',').strip('.'))
    #  Starts with 'Meet ' and the first letter of the name is either lowercase or uppercase
    elif text.startswith('Meet ') and re.match(r'[a-z][A-Z].*', text.split()[1]):
        dog_names.append(text.split()[1].strip(',').strip('.'))
    # Starts with 'Say hello to ' and the fisrt letter of the name is either lowercase or uppercase
    elif text.startswith('Say hello to ') and re.match(r'[a-z][A-Z].*', text.split()[3]):
        dog_names.append(text.split()[3].strip(',').strip('.'))
    # Starts with 'Here we have ' and the first letter of the name is either lowercase or uppercase
    elif text.startswith('Here we have ') and re.match(r'[a-z][A-Z].*', text.split()[3]):
        dog_names.append(text.split()[3].strip(',').strip('.'))
    # Contains 'named' and the first letter of the name is either lowercase or uppercase
    elif 'named' in text and re.match(r'[A-Z].*', text.split()[text.split().index('named') + 1]):
        dog_names.append(text.split()[text.split().index('named') + 1].strip(',').strip('.'))
    # Default: No name specified or other style
    else:
        dog_names.append('NaN')

#Quality #9:

# Keywords to identify a male dog
male = ['He', 'he', 'him', 'his', "he's", 'himself']
# Keywords to identify a female dog
female = ['She', 'she', 'her', 'hers', 'herself', "she's"]

# create a containair to temporary store the results
dog_gender = []

#loo through the text column
for text in df_clean['text']:
    # Find males
    if any(map(lambda v:v in male, text.split())):
        dog_gender.append('male')
    # Find females
    elif any(map(lambda v:v in female, text.split())):
        dog_gender.append('female')
    # Default: no femail or female identified
    else:
        dog_gender.append('NaN')

# Save the results to a new column called 'dog_name'
df_clean['dog_gender'] = dog_gender
